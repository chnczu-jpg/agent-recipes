# Agent Recipes 三仓库统一对标总表

日期：2026-07-11

## 大白话结论

Agent Recipes **还没有**超过 Cass、Remnic、TencentDB Agent Memory 三者的最强组合。

现在能成立的说法只有：菜谱在“候选必须审核、弱匹配敢拒绝、执行锁具体版本、结果绑定本次锁、
来源和不能声称什么都写清楚”这些收窄能力上局部领先，而且核心明显更轻。

当前不能成立的说法是：菜谱的长期记忆、上下文压缩、跨宿主接入、恢复流水线和工程
成熟度也已经超过它们。

机器可检查真值见 `THREE_REPO_COMPETITIVE_SCORECARD.json`。

## 怎么比

不是给四个项目随便打总分。每一项都取三个上游仓库里已经验证的最强能力，组成一个“虚拟最强
对手”。菜谱有一项硬短板，就不能靠其他项目赢得多来抵消。

状态只有五种：

- `ahead`：有证据领先虚拟最强对手；
- `parity`：追平收窄合同，但不夸大到对方全部能力；
- `behind`：明确落后；
- `unproven`：可能有效，但证据不够；
- `not_comparable`：法律或产品边界不允许比较。

## 全功能对标

如果用户说的“超过三者相加”是指所有功能都超过，那么当前明确不成立：

| 能力 | 当前领先者 | 菜谱决定 |
|---|---|---|
| 广义长期会话记忆 | Remnic / TencentDB | 核心不重造，当前落后 |
| 上下文压缩和 offload | TencentDB | 核心不重造，当前落后 |
| OpenClaw/Hermes 等宿主接入 | Remnic / TencentDB | 当前只有本地客户端证据，落后 |
| 长任务检查点和恢复 | TencentDB | 没有真实中断证据前不塞进核心，当前落后 |

所以项目必须选清楚承诺：

1. **合理目标**：在“安全、可审核、可执行的 Agent 经验复用”上超过三个仓库的最强组合；
2. **不合理目标**：在它们所有记忆、压缩、宿主和基础设施功能上全面超过，同时还更轻。

第二个目标自相矛盾。把所有功能都塞进来，就不可能继续保持更轻。

## 剩余一个真正阻断项

### 已关闭：fresh agent 本地真实效果门

同一批全新任务，保留无菜谱和有菜谱的原始输出。controller 只发题和评分，不能替双方写答案。
只有稳定减少重复错误、没有增加乱套，才能过。

当前真实结果不是全绿：v4 因题目范围含糊得到 0胜2平4负，只作失败证据；明确写出 SampleProject 范围的
v5 得到 3胜2平1负，并保留了一次错锁。系统随后把输掉的 PIP 题走完 correction -> review ->
原菜谱 v2，v6 又暴露“画中画”中文 no-match，修复后 v7 fresh Agent 用真实 v2 lock 正确停在
“找不到坐标就询问/恢复，禁止默认模板覆盖”。随后固定门槛的 v8 使用 6 道全新题，菜谱组
得到 5胜1平0负，5 个正例全部命中预期菜谱并创建真实锁，纯代码负例正确 no-match，错锁为 0。
因此 `fresh_agent_production_effect` 的本地窄门关闭。它仍不证明任意领域、真实剪辑执行或成片质量。

### 已关闭：固定同语料召回质量门

固定 24 条 SampleProject 菜谱和 227 条历史题库来源记录；发现 116 条重复后，只按 111 道唯一题计分。
core 正例 55/55、误召回 0；Cognee/Graphiti 投影各 53/55、误召回 3.6%；真实本地 Qwen
正例 47/55、误召回 6.3%、平均每题约 31 毫秒；15 个纯 no-match 全部拒绝。四条链都过预设门。

这只关闭固定本地比较缺口。Cognee/Graphiti 测的是同语料投影，不是完整原生记忆/图谱运行时；
也不能把 SampleProject 旧题通过说成任意新领域都准。

### P0：工程成熟度

本机能做的部分已经过：Qwen 可重复启停和被杀后恢复；0.1.0 升 0.1.1、回退 0.1.0、再升
0.1.1，事件日志字节不变；Python 3.11/3.13/3.14 三套新环境都完成 wheel 治理链和 MCP；
3.11 与 3.13 各自跑完 261 项全量测试。

GitHub Actions `29115605471` 已在 Python 3.11/3.12/3.13 跑绿，公开仓库和 `v0.1.1`
Release 也已真实存在。还没过的只剩另一台物理机器安装同一个 wheel 并复现。

## 明确不做

- 不因 TencentDB 强就把整个 offload 和长期会话记忆塞进核心；
- 不因 Remnic 功能多就复制它的多包系统；
- 不再接触 Cass 受限源码；
- 不用总测试数、代码行数或 MCP 工具数冒充产品更强；
- 不把固定 cohort 通过说成 fresh agent 已经稳定变聪明。
